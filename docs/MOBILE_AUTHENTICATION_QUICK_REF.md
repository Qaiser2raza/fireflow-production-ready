## 🚀 3-Minute Setup

### Step 1: Import useIsMobile Hook
```tsx
import { useIsMobile } from '../client/hooks/useIsMobile';

export const MyComponent = () => {
  const isMobile = useIsMobile();
  
  return isMobile ? <MobileLayout /> : <DesktopLayout />;
};
```

### Step 2: Use Responsive Classes
```tsx
// Sidebar
<aside className="hidden md:flex ...">

// Full width header on mobile
<header className="w-full md:w-[calc(100%-16rem)] ...">

// Stack on mobile, side-by-side on desktop
<div className="flex flex-col md:flex-row ...">
```

### Step 3: Fetch with Auth Token
```tsx
const response = await fetchWithAuth(`/api/endpoint`);
if (response.ok) {
  const data = await response.json();
}
```

## 🔐 Authentication Flow Commands

### Login (UI Component)
```typescript
const handleLogin = async (pin: string) => {
  const response = await fetch('/api/auth/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ pin, deviceFingerprint })
  });
  
  const { staff, tokens } = await response.json();
  
  // Store tokens
  sessionStorage.setItem('accessToken', tokens.access_token);
  sessionStorage.setItem('accessTokenExpiry', 
    (Date.now() + tokens.expires_in * 1000).toString()
  );
  
  // Update app state
  setCurrentUser(staff);
  setActiveView('DASHBOARD');
};
```

### Device Pairing (UI Component)
```typescript
const handlePairingCode = async (code: string) => {
  const response = await fetch('/api/pairing/verify', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      code,                    // "ABC123"
      deviceFingerprint,       // Generated by device
      deviceName,              // "iPhone 12"
      platform                 // "ios" or "android"
    })
  });
  
  const { auth_token } = await response.json();
  
  // Store auth token
  localStorage.setItem('deviceAuthToken', auth_token);
  
  // Auto-login
  handleLogin(savedPin);
};
```

### API Call with Auth
```typescript
// OLD ❌
const response = await fetch('/api/orders');

// NEW ✅
const response = await fetchWithAuth('/api/orders');

// fetchWithAuth automatically:
// 1. Includes Authorization header with token
// 2. Checks token expiry
// 3. Refreshes if needed
// 4. Retries on 401
```

## 📱 Common Mobile Patterns

### Sidebar Hiding
```tsx
// Desktop: visible, Mobile: hidden
<aside className="hidden md:flex w-64 ...">
           ↑
     Mobile breakpoint (768px)
```

### Full-Width Content
```tsx
// Mobile: takes 100% width
// Desktop: restricted by sidebar
<main className="flex-1 ...">
```

### Touch-Friendly Buttons
```tsx
// Minimum 44px height for touch targets
<button className="h-11 ...">  // 44px = 11 * 4px
<button className="py-3 ...">  // At least 24px + padding
```

### Status Indicators
```tsx
// Show/hide based on viewport
<div className="hidden md:block">
  {/* Desktop only */}
</div>

<div className="md:hidden">
  {/* Mobile only */}
</div>
```

## 🔍 Debugging Mobile Issues

### Check Token
```typescript
// Browser console
sessionStorage.getItem('accessToken')
sessionStorage.getItem('accessTokenExpiry')

// Check if expired
new Date(parseInt(sessionStorage.getItem('accessTokenExpiry') || '0'))
```

### Check Mobile Detection
```typescript
// Browser console
window.innerWidth < 768  // Should match useIsMobile() state
window.matchMedia('(max-width: 767px)').matches
```

### Check API Headers
```
DevTools → Network tab
Click any API request
Headers tab → Request Headers
Look for: Authorization: Bearer <token>
```

### Check Layout
```
DevTools → Elements tab
Find <aside> element
Check classes: "hidden md:flex" or "hidden"
Check computed styles: display: none or display: flex
```

## ⚡ Performance Tips

### Don't Do This ❌
```typescript
// ❌ Runs on every render
activeView === 'POS' 
  ? (window.innerWidth < 768 ? <Mobile /> : <Desktop />) 
  : null
```

### Do This Instead ✅
```typescript
// ✅ State-based, runs only on threshold crossing
const isMobile = useIsMobile();
useEffect(() => {
  // Runs only when isMobile changes
}, [isMobile]);
```

### Memoize Components
```typescript
// Prevent unnecessary re-renders
const MobileLayout = React.memo(({ data }) => {
  return <div>...</div>;
});
```

## 📊 Token Management

### Check Expiry
```typescript
const isTokenExpired = () => {
  const expiry = sessionStorage.getItem('accessTokenExpiry');
  return expiry && Date.now() > parseInt(expiry);
};

// Use before making API calls
if (isTokenExpired()) {
  console.log('Token expired, will auto-refresh');
}
```

### Manual Refresh
```typescript
const refreshToken = async () => {
  const response = await fetch('/api/auth/refresh', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      refresh_token: sessionStorage.getItem('refreshToken')
    })
  });
  
  const { access_token, expires_in } = await response.json();
  
  sessionStorage.setItem('accessToken', access_token);
  sessionStorage.setItem('accessTokenExpiry',
    (Date.now() + expires_in * 1000).toString()
  );
};
```

## 🚨 Emergency Fixes

### Token stuck/expired?
```typescript
// Clear all auth state
sessionStorage.removeItem('accessToken');
sessionStorage.removeItem('refreshToken');
sessionStorage.removeItem('accessTokenExpiry');
localStorage.removeItem('deviceAuthToken');

// Redirect to login
window.location.href = '/login';
```

### Sidebar stuck visible on mobile?
```
1. Check: <aside className="hidden md:flex">
          Element has "hidden" class
2. Hard refresh: Ctrl+Shift+R
3. Rebuild CSS: npm run build:css
```

### API calls failing with 401?
```typescript
// 1. Check token exists
console.log(sessionStorage.getItem('accessToken'));

// 2. Check not expired
console.log(Date.now() < parseInt(sessionStorage.getItem('accessTokenExpiry') || '0'));

// 3. Try manual refresh
refreshToken();

// 4. If all fails, logout and re-login
logout();
```
